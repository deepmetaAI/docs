# Kling（可灵）视频生成

Cloubic 目前已支持对 Kling（可灵）视频大模型的统一接入与调用。通过本章节提供的接口说明，您可以在同一套 API 下发起视频生成请求，调用 Kling 视频生成接口生成视频。支持 V2 和 V3 版本，V3 版本新增多镜头视频生成、主体库和视频续写等高级功能。{{ className: 'lead' }}

## 文生视频

<Row>
  <Col>
    <Endpoint method="POST" path="https://api.cloubic.com/v1/video/generations" />

    ### 请求头

    <Properties>
      <Property name="Authorization" type="string" required>
        Bearer Token，格式：`Bearer YOUR_API_KEY`
      </Property>
      <Property name="Content-Type" type="string" required>
        请求体格式，固定为 `application/json`
      </Property>
    </Properties>

    ### 请求参数

    <Properties>
      <Property name="model" type="string" required>
        V2 版本：`kling-v2-5-turbo-std`、`kling-v2-5-turbo-pro`、`kling-v2-6`
        
        V3 版本：`kling-v3-std`、`kling-v3-pro`
      </Property>
      <Property name="prompt" type="string" required>
        文本提示词，生成视频的文本描述（与 `metadata.multi_prompt` 互斥）
      </Property>
      <Property name="duration" type="integer">
        视频时长参数（秒）
        
        V2：可选值 5、10，默认 `5`
        
        V3：可选值 3-15，使用 `multi_prompt` 时需与各镜头时长之和相等
      </Property>
      <Property name="size" type="string">
        分辨率参数，输出分辨率，格式为宽度x高度，默认 `16:9`
        <Options>
          <Option value="16:9" />
          <Option value="1:1" />
          <Option value="9:16" />
        </Options>
      </Property>
      <Property name="metadata" type="object">
        扩展参数（JSON 对象格式），V3 版本支持多镜头等高级功能
      </Property>
    </Properties>
  </Col>

## V3 多镜头视频生成

Kling V3 版本支持多镜头视频生成功能，可以通过 `metadata` 参数配置多个镜头的提示词和时长。多镜头模式支持文生视频和图生视频两种方式。

<Row>
  <Col>
    ### metadata 参数说明（V3 专用）

    <Properties>
      <Property name="multi_shot" type="boolean" required>
        是否生成多镜头视频
        
        - `true`：启用多镜头模式，`shot_type` 和 `multi_prompt` 生效，`prompt` 参数无效
        - `false`：单镜头模式，使用 `prompt` 参数
        
        注意：多镜头模式不支持设定首尾帧生视频（Pro 版 `image_list` 功能）
      </Property>
      <Property name="shot_type" type="string">
        分镜方式，当 `multi_shot` 为 `true` 时必填
        <Options>
          <Option value="customize">自定义分镜</Option>
          <Option value="intelligence">智能分镜</Option>
        </Options>
      </Property>
      <Property name="multi_prompt" type="array">
        各分镜信息数组，包含提示词、时长等
        
        - 最多支持 6 个分镜，最少 1 个分镜
        - 每个分镜提示词最大长度 512 字符
        - 每个分镜时长不小于 1 秒，不大于总时长
        - 所有分镜时长之和必须等于 `duration` 参数
        
        每个分镜对象包含：
        - `index`（integer）：分镜序号
        - `prompt`（string）：分镜提示词
        - `duration`（integer）：分镜时长（秒）
      </Property>
      <Property name="image" type="string">
        参考图片 URL（可选），支持多镜头图生视频
        
        提供图片后，多镜头视频将基于该图片进行生成
      </Property>
      <Property name="images" type="array">
        参考图片数组（可选），支持多张图片输入
        
        可与多镜头模式结合使用，实现基于图片的多镜头视频生成
      </Property>
    </Properties>

    ### V3 Pro 版本额外参数

    <Properties>
      <Property name="image_list" type="array">
        首尾帧图片列表，仅 `kling-v3-pro` 支持
        
        每个对象包含：
        - `image_url`（string）：图片 URL
        - `type`（string）：图片类型，可选值：`first_frame`（首帧）、`end_frame`（尾帧）
        
        注意：不支持与多镜头模式同时使用
      </Property>
      <Property name="element_list" type="array">
        参考主体列表，基于主体库中主体的 ID 配置
        
        - 最多支持 3 个参考主体
        - 主体分为视频角色主体和多图主体，适用范围不同
        
        每个对象包含：
        - `element_id`（long）：主体库中的主体 ID
      </Property>
      <Property name="video_list" type="array">
        视频续写列表，用于视频延续生成
        
        每个对象包含：
        - `video_url`（string）：参考视频 URL
        - `type`（string）：续写类型
      </Property>
    </Properties>
  </Col>

### Go 示例（V3 多镜头）

```go
package main

import (
    "bytes"
    "encoding/json"
    "fmt"
    "io"
    "net/http"
)

type MultiPrompt struct {
    Index    int    `json:"index"`
    Prompt   string `json:"prompt"`
    Duration int    `json:"duration"`
}

type Metadata struct {
    MultiShot   bool          `json:"multi_shot"`
    ShotType    string        `json:"shot_type"`
    MultiPrompt []MultiPrompt `json:"multi_prompt"`
}

type VideoRequest struct {
    Model    string   `json:"model"`
    Duration int      `json:"duration"`
    Size     string   `json:"size"`
    Metadata Metadata `json:"metadata"`
}

func main() {
    reqBody := VideoRequest{
        Model:    "kling-v3-std",
        Duration: 5,
        Size:     "16:9",
        Metadata: Metadata{
            MultiShot: true,
            ShotType:  "customize",
            MultiPrompt: []MultiPrompt{
                {Index: 1, Prompt: "长翅膀的猫", Duration: 2},
                {Index: 2, Prompt: "长翅膀的猫原地转圈", Duration: 3},
            },
        },
    }

    jsonData, _ := json.Marshal(reqBody)
    req, _ := http.NewRequest("POST",
        "https://api.cloubic.com/v1/video/generations",
        bytes.NewBuffer(jsonData))
    req.Header.Set("Content-Type", "application/json")
    req.Header.Set("Authorization", "Bearer YOUR_API_KEY")

    client := &http.Client{}
    resp, _ := client.Do(req)
    defer resp.Body.Close()

    body, _ := io.ReadAll(resp.Body)
    fmt.Println(string(body))
}
```

## 注意事项

1. **多镜头限制**：
   - 最多支持 6 个分镜，最少 1 个分镜
   - 所有分镜时长之和必须等于总时长
   - 每个分镜提示词最大 512 字符
   - 多镜头模式不支持首尾帧控制（Pro 版 `image_list` 功能）
   - 多镜头模式支持图片输入，可通过 `image` 或 `images` 参数实现图生视频

2. **V3 Pro 专属功能**：
   - 首尾帧控制：通过 `image_list` 指定首帧和尾帧图片（不支持多镜头模式）
   - 主体库：最多支持 3 个参考主体
   - 视频续写：基于已有视频进行延续生成

3. **参数互斥关系**：
   - `prompt` 与 `metadata.multi_prompt` 互斥
   - 多镜头模式（`multi_shot: true`）时，`prompt` 参数无效
   - 单镜头模式（`multi_shot: false`）时，`shot_type` 和 `multi_prompt` 参数无效
   - Pro 版 `image_list`（首尾帧）不支持与多镜头模式同时使用

4. **图片要求**：
   - 支持 URL 和 Base64 格式
   - 图片 URL 必须可公开访问
   - 建议图片分辨率与输出视频比例一致
   - 多镜头模式可以使用 `image` 或 `images` 参数进行图生视频
